# Contributing to FreeViewer

Thank you for your interest in contributing to FreeViewer! This document provides guidelines for contributing to the project.

## 🎯 How Can I Contribute?

### Reporting Bugs
- Use GitHub Issues to report bugs
- Include your OS, Python version, and error messages
- Provide steps to reproduce the issue
- If possible, include screenshots

### Suggesting Features
- Check existing issues first to avoid duplicates
- Explain the use case and benefits
- Consider implementation complexity

### Code Contributions
1. Fork the repository
2. Create a feature branch
3. Write clean, commented code
4. Test your changes thoroughly
5. Submit a pull request

## 📝 Code Style

- Follow PEP 8 guidelines
- Use meaningful variable names
- Add comments for complex logic
- Keep functions focused and small
- Use type hints where appropriate

## 🧪 Testing

- Test on your local machine before submitting
- If possible, test on different operating systems
- Verify both LAN and internet connections
- Check file transfer functionality

## 🔍 Areas We Need Help

### Critical Needs
1. **NAT Traversal** - Solving P2P connection issues
2. **Cross-Platform Testing** - Linux and macOS compatibility
3. **Documentation** - Improving setup guides

### Feature Development
- Audio streaming
- Multi-monitor support
- Clipboard sync
- Session recording
- Mobile app versions

## 📋 Pull Request Process

1. Update README.md with details of changes if needed
2. Update requirements.txt if adding dependencies
3. Reference any related issues in your PR description
4. Be responsive to code review feedback
5. Ensure CI/CD checks pass

## ⚖️ Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help newcomers feel welcome
- Credit others for their work

## 🚀 Getting Started

1. Read the main README.md thoroughly
2. Set up your development environment
3. Look for issues labeled `good first issue`
4. Ask questions in issue discussions

## 💡 Development Tips

- Use virtual environments for dependencies
- Keep your fork up to date with the main repository
- Write meaningful commit messages
- Break large changes into smaller PRs

## 📫 Questions?

- Open a GitHub Discussion for general questions
- Comment on specific issues for clarification
- Tag maintainers if you need urgent help

## 🙏 Recognition

All contributors will be recognized in the project's README.

Thank you for helping make FreeViewer better!
